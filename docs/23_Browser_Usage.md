Browser Usage Rule for Debugging and Testing

Whenever debugging, testing, or working in Chrome is required, Anti-Gravity must always use a separate, unused browser window so that the user’s active work is never interrupted.

Follow this process strictly:

Never use or interact with any Chrome window that appears to be actively in use.

Identify a Chrome window that is completely unused (no active websites or working tabs open).

If an unused window exists, select that single window and perform all debugging and testing exclusively within it.

If no unused Chrome window exists, open a brand-new Chrome window and use only that window.

Once selected, all browser-based debugging and testing must remain confined to that one dedicated window.

Do not switch between multiple windows or use the user’s active browsing sessions at any time.

The purpose of this rule is to ensure that debugging and development work never interferes with the user’s current browsing, workflows, or active sessions.
